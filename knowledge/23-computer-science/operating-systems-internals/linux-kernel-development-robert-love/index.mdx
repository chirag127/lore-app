## Overview

*Linux Kernel Development* (July 2010, 3rd ed.) by Robert Love is the most authoritative and readable guide to Linux kernel internals in print. Written by a kernel contributor with over a decade of hands-on involvement — including the preemptive kernel, process scheduler, and inotify — it bridges the gap between "I read about the kernel once" and "I just submitted a patch to mainline."

The book covers the Linux 2.6.3x kernel series across 16 chapters: process model, CFS scheduling, system calls, kernel data structures (new in 3rd ed.), interrupt handlers and bottom halves (new in 3rd ed.), kernel synchronization, memory management, the VFS, filesystems, procfs, networking, and kernel debugging. Every concept is grounded in real kernel source: Love cites `task_struct`, `sched_entity`, `struct page`, `inode`, `dentry`, `bio`, and `sk_buff` — the hundred-odd structures that make up daily kernel life.

## Robert Love

**Robert Love** is among the most trusted voices in the Linux community. His kernel contributions span the process scheduler, the preemptive kernel, the kernel events (kevent) layer, inotify, and VM enhancements. At Google, he was a member of the engineering team that built the kernel for the Android mobile platform. Before Google he served as Chief Architect for Linux Desktop at Novell; before that as a kernel engineer at MontaVista Software and Ximian.

He is a contributing editor for *Linux Journal*, a regular conference speaker, and the author of *Linux System Programming* and *Linux in a Nutshell*. Love writes not as an academic describing a system from outside, but as a long-time contributor who knows which structs matter and which API edges are sharp.

## Why This Book Matters

Before Love, the best Linux kernel book for developers was Bovet & Cesati's *Understanding the Linux Kernel* — readable, but heavily x86-specific and more about reading the kernel than writing for it. *Linux Kernel Development* took a different approach: it teaches the kernel as a set of subsystems a developer might need to extend, optimize, or debug.

That decision — to write for the contributor, not just the student — is what made this book indispensable. Within months of the first edition's release, attendees at Linux conferences were citing Love's explanation of `task_struct` and `clone()` as the clearest available anywhere. The 3rd edition extended that reputation to the VFS, memory, and synchronization chapters.

The book remains the recommended starting point for anyone serious about Linux kernel work, and its chapter on submitting patches is still the clearest practical guide to becoming a recognized mainline contributor.
