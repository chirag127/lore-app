## Introduction

Welcome to BookAtlas. Today: *Effective Python* by Brett Slatkin.
Second edition, 2019. 90 specific ways to write better Python.

This is not a book for beginners. If you do not know Python syntax,
start elsewhere. This book assumes you can write Python. It teaches
you to write *good* Python.

---

## The Approach

Slatkin is a Google engineer. His approach is pragmatic: each of
the 90 items presents a specific practice, explains why it works,
and shows before-and-after code.

Item 1: "Know Which Version of Python You're Using." You would
think this is obvious. But you would be surprised how many people
skip it.

Item 27: "Use @classmethod Polymorphism to Construct Objects
Generically." This is the kind of pattern that separates
intermediate from advanced Python.

---

## The Highlights

Some items stand out:
- Item 35: "Causes for Generators" — explains why generators
  matter more than most Python developers realize
- Item 51: "Combine asyncio with Threading for Blocking I/O" —
  practical guidance for real-world async programming
- Item 72: "Concurrent.futures for the Win" — the simplest path to
  parallelism

---

## The Verdict

*Effective Python* is not a fun read. It is not a page-turner. It
is a tool. If you write Python professionally, you need it on
your shelf. Read a few items every week. Apply them. Repeat.

**Rating: 9.0/10** — The best intermediate Python book. Indispensable.
