## Overview

*Effective Python: 90 Specific Ways to Write Better Python* (2nd ed.,
2019) by Brett Slatkin is a reference-style guide for intermediate
to advanced Python developers. Each of the 90 items covers a
specific best practice, pattern, or pitfall in idiomatic Python.

---

## Executive Summary

Slatkin focuses on practical Pythonic approaches — the patterns that
experienced Python developers use instinctively. The book covers
everything from basic data structures to metaprogramming, concurrency,
and testing.

| Knowledge Level | What You Get |
|---|---|
| Beginner | Solid foundation in Pythonic basics |
| Intermediate | Deep understanding of idioms and patterns |
| Advanced | Expert techniques and performance optimization |

---

## Key Takeaways

1. **Prefer simple over clever.** Python's readability is its
   greatest strength. Code is written once and read many times.

2. **Know your standard library.** Python's built-in modules
   (itertools, collections, functools) solve most problems.

3. **Use generators for large data.** Memory efficiency matters.
   Generators and iterators process streams without loading all data.

4. **Type annotations improve maintainability.** Python is dynamic
   but annotations catch bugs and serve as documentation.

5. **Understand concurrency models.** Threads, asyncio, and
   multiprocessing each have distinct use cases. Pick the right one.

6. **Test everything.** Properties, fixtures, and parameterized
   tests make testing fast and comprehensive.

## Why This Book Matters

Most Python books teach syntax. *Effective Python* teaches judgment.
It bridges the gap between "I can write Python" and "I write good
Python." It is the book you read after the tutorial.
