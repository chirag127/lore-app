# Software Design & Architecture

## Overview

Software design and architecture form the bridge between code and system. This subcategory covers the foundational texts on design patterns (named, reusable solutions to recurring problems), architectural styles (clean, hexagonal, layered), and the principles that guide both. Books here address how to structure software for maintainability, testability, and long-term value — from the class level to the system level.
