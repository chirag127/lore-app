# Software Design Principles

## Overview

Design principles guide how software is organized internally — how
responsibilities are allocated, how modules communicate, and how change can
be isolated. This subcategory covers object-oriented design principles
(SOLID), design patterns (Gang of Four patterns, their intent and
applicability), domain-driven design (ubiquitous language, aggregates,
bounded contexts), and advanced patterns like event sourcing and CQRS.
