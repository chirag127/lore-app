## 3.1 Linear Basis Function Models

A general linear model applies fixed nonlinear **basis functions** `ПҶ_j(x)` to the input:

```
y(x, w) = ОЈ_{j=0}^{M-1} w_j ПҶ_j(x) = w^T ПҶ(x)
```

Common basis functions:

| Basis | Formula |
|-------|---------|
| Polynomial | `ПҶ_j(x) = x^j` |
| Gaussian | `ПҶ_j(x) = exp(-(x-Ој_j)ВІ/(2sВІ))` |
| Sigmoidal | `ПҶ_j(x) = Пғ(x - Ој_j)` |
| Fourier | `ПҶ_j(x) = sin/cos(Пү_j x)` |

## 3.2 Maximum Likelihood and Least Squares

Assuming Gaussian noise `N(t | y(x,w), ОІвҒ»В№)`:

```
E(w) = ВҪ ОЈ (t_n - w^T ПҶ_n)ВІ
w_ML = (ОҰ^T ОҰ)^(-1) ОҰ^T t
```

**Bias-variance dilemma**: as `M` (basis functions) increases, variance of `w_ML` increases while bias decreases.

## 3.3 Regularized Least Squares (Ridge Regression)

Adding a **quadratic regularizer** (equivalent to Gaussian prior on `w`):

```
E(w) = ВҪ ||t - ОҰw||ВІ + (О»/2) ||w||ВІ
w_MAP = (О»I + ОҰ^T ОҰ)^(-1) ОҰ^T t
```

This is **ridge regression**. The parameter `О»` controls model complexity. Bishop also discusses **automatic relevance determination** (ARD) as a Bayesian alternative to choosing `О»` by cross-validation.

## 3.4 Bayesian Linear Regression

```mermaid
flowchart LR
    Prior["Prior over w:<br/>N(m_N, S_N)"] --> Post["Posterior:<br/>N(m_N, S_N)"]
    Data["Data D = {X, t}"] --> Post
    Post --> Pred["Predictive:<br/>Student-t"]
    Sub1["m_N = ОІ S_N ОҰ^T t<br/>S_N = ОұI + ОІОҰ^TОҰ"] --> Post
    Sub2["Оұ = prior precision<br/>ОІ = noise precision"] --> Sub1
```

The Bayesian approach places a prior over `w` (usually zero-mean Gaussian with precision `Оұ`), then computes the posterior `p(w | D)`. The predictive distribution for a new input `x` integrates over `w`:

```
p(t | x, D) = вҲ« p(t | x, w) p(w | D) dw = St(t | Ој, ПғВІ, ОҪ)
```

The predictive distribution is a **Student-t** distribution вҖ” robust to outliers.

## 3.5 Evidence Framework

The **evidence function** `p(D | Оұ, ОІ)` is maximized to set hyperparameters `Оұ` and `ОІ`, without cross-validation:

```
ln p(D | Оұ, ОІ) = -E_D - E_W + const
```

where `E_D` is the data-fit term and `E_W` is the complexity penalty controlled by the effective number of parameters `Оі`.

**Effective number of parameters**: `Оі = ОЈ_i О»_i / (О»_i + Оұ)`, where `О»_i` are eigenvalues of `ОІ ОҰ^T ОҰ`.

## 3.6 Limitations and Extensions

- Linear in parameters, but nonlinear via basis functions
- Scaling to large `M` requires iterative methods (N-gradient)
- **Sparse Bayesian Learning** (Tipping 2001) automatically drives some `Оұ_j вҶ’ вҲһ` to prune irrelevant basis functions
